Utaka

Purpose and Beginnings

The primary purpose behind Utaka is to create a simple cloud storage service that is API compatible with Amazon's Simple Storage Service (S3). Humanitarian International Services Group was looking for a way for its members to upload files to a central repository and allow other members to view and edit these files. S3 seemed like a good option, but the development team wanted to host their own service. Making Utaka API compatible had a dual benefit here. One, we could use other premade clients and libraries with little modification, and two, if we ever outgrow our own service, we can switch to S3 by changing only a few lines of code. The development team leader was also interested in making the project open source, to allow others to help with the project and set up there own cloud storage servers, so the project has been placed here on GitHub.

Goals

  • To create a simple and stable cloud storage service.
  • To make a well documented and easy to understand open-souce project.

What Utaka Doesn't Do

Currently Utaka is only in the alpha stage, so it is obviously not complete. But we are planning on not implementing some features from S3 initially:

  • SOAP API
  • Bucket Locations (these are simply ignored, attempting to use them doesn't cause errors)
  • Torrents
  • Requester Pays Buckets (since you host your own service no one pays)

What Utaka Does Differently

  • Logging (Only best-effort on S3, when activated Utaka will log every request and deliver the logs at a regular interval)
