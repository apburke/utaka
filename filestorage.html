Utaka File Storage

Location

Utaka can store its files almost anywhere you want. Local folders, external drive, NFS storage, or anything you can give a path to can be used. The .utaka_config file under utaka/config/ contains an option for the filesystem path. All utaka disk access will be limited to that path.

Organization

Each bucket is given its own folder under the master path, and each object that will be stored in a bucket is placed in that bucket's folder. However, that object is not given its key for a name. A SHA1 hash is generated for each object as its filename, and the first nine characters of the hash are also used to create three three-character folders. For example, say a user sends an object with key testfile.mp3 to bucket music. Utaka will generate a SHA1 hash c85d23111b1b073d01af791da95cb2ac005df282 for the object. It then generates a path (Utaka File Storage Path)/music/c85/d23/111/ for the file and creates any new folders in the path. Then it creates the file with the filename c85d23111b1b073d01af791da95cb2ac005df282. The entire path to the file will then be (Utaka File Storage Path)/music/c85/d23/111/c85d23111b1b073d01af791da95cb2ac005df282.
